### Explorer

- This panel is the **file-first view** of the workspace.
- Open `brief.md`, `status.md`, `RUN.md`, or `RESULT.json` to see what the run has already produced.
- In a real project, this is where you verify that the graph and the text trace are backed by durable files.
- `FILES` shows the current active workspace root, which is not always the quest root; manually created terminal files need a refresh, and files in another worktree will not automatically appear in the current view.
